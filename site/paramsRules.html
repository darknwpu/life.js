• 两种方式

    1. 枚举式，参数个数不超过三个时，允许使用该方式，但必须同时支持对象式。
    2. 对象式，参数名统一为：options。只要有参数就得支持该方式，当参数不超过三个时，可以在组件内部实现枚举式

  • 参数设计规范

    1. 使用驼峰命名法，语义清晰
    2. 参数顺序：必填参数 ==》 选填参数，属性参数 ==》 方法参数（callback||onXxx）。
    3. 参数单层级，除了参数 data 外，尽量使参数保持在一个层级。

  • 常用参数

    1. id,关联的 DOM 节点 ID，当组件只跟一个 DOM 节点关联时，使用该参数
    2. class,关联的 DOM 节点 Class。当组件和多个 DOM 节点关联时，使用该参数
    3. exClass, UI 组件附加的 Class，为增加 UI 组件的定制性可实现该接口
    4. exParams, 当参数可作为回调函数或关联函数的扩展参数
    5. type,当组件只持多种模式时，可使用该参数
    6. theme,当组件只持多种主题风格时，可使用该参数
    7. skin,当组件只持多种颜色的皮肤时，可使用该参数
    8. onXxx,组件支持的事件
    9. isXxx,开启或关闭某项功能
    10. callback,回调函数，注意与onXxx事件的区别
    11. funName,绑定的函数名，相应函数必须定义在可调用的相同作用域或在全局作用域中
    12. width,设置组件的宽
    13. height,设置组件的高
    14. align,设置组件的对齐方式
    15. zIndex,设置组件的层级
    16. timeout,设置组件的过期时间
    17. left,设置组件的位置 left
    18. top,设置组件的位置 top
    19. data,组件要用到的数据，array|json 格式
    20. template,组件内允许自定义的 html 模板
    21. text,文本、字符串
    22. number,数字
    23. url,链接
    24. animate,切换效果
    25. direction,方向
    26. interval,间隔时间
